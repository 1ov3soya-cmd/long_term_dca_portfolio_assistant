# 长期定投组合助手（Long-term DCA Portfolio Assistant）

这是一个面向中国 A 股 / 场内 ETF 场景的长期定投组合助手，核心定位是：

- 长期
- 低频
- 可解释
- 防大错

它不是短线交易机器人，也不是自动下单系统。当前版本只做以下事情：

- 用 `e-finance` 优先拉取 ETF / 股票日线历史行情
- 将行情缓存到本地 CSV
- 每月第 1 个交易日生成定投建议
- 每周最后 1 个交易日做风险巡检
- 按 `GREEN / YELLOW / RED` 输出风险状态
- 通过低频 MVP 回测器评估长期定投效果
- 输出 CSV / Markdown 报告，供人工确认后执行

## 一、当前实现状态

- 已完成：配置驱动、provider 抽象、本地缓存、月度建议、红线监控、MVP 回测器、报告输出、东财 skill 占位、基础测试。
- 已预留：`QlibAdapter`。
- 未深接：Qlib 数据准备与正式回测适配。

当前回测引擎是“先 MVP 后 Qlib”的过渡方案：

- 当前默认：自研低频 MVP 回测器
- 预留接口：`src/providers/qlib_adapter.py`
- 原因：先保证长期定投主流程可跑通，再逐步接入 Qlib，避免 Qlib 数据结构问题阻塞整个项目

## 三、当前默认投资框架

### 1. 资产配置

- ETF：80%
- 股票：20%

这两个比例是当前核心约束，不应在未经人工确认的情况下修改。

### 2. ETF 池

- `510300`：30%
- `510500`：20%
- `515180`：20%
- `518880`：10%

默认原因：

- 先用明确、固定、可解释的池子做核心底仓
- 不自动扩池，避免第一版把精力放在“不断找新标的”上

### 3. 股票池

- `600519`
- `000858`
- `600036`
- `000333`
- `601318`

默认原因：

- 初版股票池已由人工指定
- 最多持有 5 只，每只目标权重 4%
- 若触发红线，只暂停买入或进入复核，不自动替换

## 四、关键配置与默认值说明

所有关键参数都在 `config/` 下，关键假设不会写死在代码里。

### 1. 执行频率

- 月度正式定投：每月第 1 个交易日
- 周度巡检：每周最后 1 个交易日
- 日度：仅更新数据，不生成交易决策

默认原因：

- 符合长期定投和低频决策要求
- 降低噪声与过度交易风险

### 2. 风险红线

ETF：

- `YELLOW`：相对参考高点回撤 >= 15%，或跌破长期均线后持续弱势
- `RED`：相对参考高点回撤 >= 25%

股票：

- `YELLOW`：相对成本回撤 >= 12%
- `RED`：相对成本回撤 >= 20%，或 `thesis_broken=true`

组合：

- 当前组合回撤 >= 20% 时触发组合层 `RED`

默认原因：

- ETF 主要承担底仓功能，红线更多用于暂停加仓与人工复核
- 股票部分波动更大，因此用相对成本的更严格阈值
- 组合层红线用于防止整体风险失控

### 3. 参考高点定义

当前版本的 ETF “参考高点”默认配置为：

- `reference_high_mode: expanding_peak`

含义：

- 使用截至当前时点的历史累计高点作为参考高点
- 这是第一版最简单、最透明的做法

如果未来你希望改成“近 250 日高点”或其他窗口方式，应先在配置中明确。

### 4. 交易成本与成交规则

ETF：

- 买入佣金：0.03%
- 卖出佣金：0.03%
- 卖出印花税：0
- 滑点：0.10%

股票：

- 买入佣金：0.03%
- 卖出佣金：0.03%
- 卖出印花税：0.10%
- 滑点：0.20%

其他：

- 最小交易单位：100 股
- 若资金不足整手则向下取整
- 若不可交易则不成交并保留现金

重要说明：

- 以上仅为 MVP 保守假设，不代表真实成交结果
- 第一版未实现涨跌停与停牌的精细撮合逻辑，只保留了接口与限制说明

## 五、卖出逻辑边界

当前版本**不自动卖出**。

设计原因：

- 本项目目标是“长期定投 + 风险提醒 + 人工确认执行”
- 若第一版引入自动卖出，容易滑向短线止损系统，与项目定位冲突

当前代码中的卖出相关处理原则是：

- `YELLOW`：暂停新增买入，人工复核
- `RED`：必须人工复核
- 是否减仓或退出：由人工确认后执行

代码中已用注释明确写出：卖出接口保留给未来“人工确认 + 明确触发条件”的版本，不在当前 MVP 自动执行。

## 六、e-finance 数据说明

当前默认主数据源是 `e-finance`。

实现策略：

- 通过 `providers/base_provider.py` 做抽象
- `providers/efinance_provider.py` 负责历史行情获取
- 返回字段统一标准化为：
  - `date`
  - `symbol`
  - `asset_type`
  - `open`
  - `high`
  - `low`
  - `close`
  - `volume`
  - `amount`

当 `e-finance` 字段变化或结构异常时，当前实现不会静默修复，而是：

1. 打印字段诊断
2. 输出样例行
3. 抛出异常
4. 等待人工确认后再修改映射

兼容扩展方案：

- `csv_provider.py`
- 未来可扩展 `akshare / tushare / 本地 CSV / parquet`

## 七、如何运行

建议先安装依赖：

```bash
pip install -r requirements.txt
```

### 1. 运行离线 demo

```bash
python -m src.main demo
```

作用：

- 使用合成 demo 数据跑通建议与回测
- 不依赖真实行情接口
- 适合先验证工程链路是否正常

### 2. 更新真实数据

```bash
python -m src.main update-data
```

### 3. 生成月度建议

```bash
python -m src.main suggest
```

### 4. 运行 MVP 回测

```bash
python -m src.main backtest
```

## 八、报告输出

月度报告输出到：

- `reports/monthly/`

回测报告输出到：

- `reports/backtest/`

至少包括：

- 当前持仓
- 本月预算
- 本月建议买入
- 当前权重 vs 目标权重
- 黄线 / 红线状态
- 人工复核事项
- 回测净值与绩效指标

## 十、当前限制

- 当前未深接 Qlib
